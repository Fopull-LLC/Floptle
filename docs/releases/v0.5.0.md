# Floptle v0.5.0 — "Loose Parts"

Animate any model — even one you never rigged. A model imported as a bunch of
separate meshes (the N64 / "loose parts" way — a body split into a head, a torso,
arms and legs) is now a first-class, fully pose-able thing: every object shows up,
you can parent objects to each other, keyframe them, and hang soft bone-chains off
the ones that should flow. Plus a Mirror-apply tool that finishes a Blender model
whose Mirror modifier never got applied, so you can skip the last round-trip through
Blender entirely.

## Every model is an object tree

- **Objects are now addressable.** A glTF with more than one mesh object used to be
  flattened into an anonymous per-material blob on import — the individual pieces
  were gone. Now the engine keeps the node tree: each mesh object stays named and
  separate, exactly like a rig's bones (they're the same thing under the hood — a
  node you can pose). A lone single-mesh prop still takes the cheap flattening path.
- **Objects vs Bones, side by side.** Select a model and the Inspector's new
  **◈ Objects & Rig** section lists its **Objects** (mesh sub-objects — ◈) and its
  **Bones** (rig joints — 🔗) in two groups. The Hierarchy tree shows the same split
  with matching icons. Click any entry to select it, then move it with the gizmo or
  keyframe it in the Animating tab — an unrigged model animates cleanly, no skinning
  required.

## Parent objects to each other

- A **"under ▸"** dropdown on each object re-parents it *within the model* — parent a
  forearm under a shoulder and posing the shoulder now carries the forearm. It's
  non-destructive: the parenting lives in a `<model>.rig.ron` sidecar beside the
  file, is re-applied on import, and keeps the object visually in place (no jump).
  Cycles are refused automatically.

## Mirror-apply — finish a model without going back to Blender

- **⇋ Apply Mirror → new .glb** completes a model whose Mirror modifier was left
  unapplied at export (so half the geometry is missing). Working from the geometry
  alone, per object it:
  - **welds** a centerline half (a head, a torso — geometry cut at the mirror plane)
    into one whole object;
  - **splits** a lateral limb (an arm, a leg — sitting fully off the plane) into a
    mirrored **`.L` / `.R`** pair you can animate independently;
  - **leaves alone** anything already symmetric across the plane.
  The result is written beside the source as `<name>.mirrored.glb` (non-destructive)
  with a Console report of exactly what it did — assign the new model and you have a
  plain, normal, both-sided character.

## Flow-rig — make hair (or cloth, or a tail) bend

- **🦴 Rig selected object to flow** drops a soft vertical bone-chain down the
  selected object and auto-weights it (a smooth top-to-bottom blend), baked into a
  new `<name>.rigged.glb`. Pose or keyframe the chain to make hair sweep, cloth
  drape, or an antenna wobble — the one object gets true vertex skinning while the
  rest of the model rides along unchanged.

## Editable pivots — set where each object rotates

- Every object gets a **rotation pivot** (its "joint"), defaulting to the object's
  geometry center — so a limb baked at the model origin no longer swings around the
  feet. Place the exact joint from the Inspector's **⌖ Pivot** fields, or flip on
  **✥ drag in view** and drag the gizmo straight to the elbow, then pose around it.
  Pivots are per-model and non-destructive (saved in the model's `.rig.ron` sidecar);
  they change only how a pose composes, never what's keyframed.

## Animating polish & stability

- **Author a clip on any rigged model, no controller required** — the **✚ New…**
  button now appears for any model in the Animating tab and creates a standalone clip
  bound to it, so you can start keying an imported character immediately.
- **The scene and the animator stop fighting.** While an object/bone is selected,
  Undo/Redo act on the *clip* (not the scene), and Delete / copy / paste / duplicate
  no longer touch scene nodes — so you can't accidentally undo the world or delete the
  thing you're animating.
- **Deleting an in-use asset is safe.** Removing a model or clip that's being animated
  now cleanly drops it everywhere instead of thrashing (the freeze some hit is gone),
  and Mirror-apply is idempotent — re-running it never stacks duplicate limbs.

## Under the hood

- **A real `.glb` writer.** The engine can now *export* glTF binary, not just import
  it — geometry, UVs, vertex color, embedded PNG textures, node hierarchy, and skins
  with inverse-bind matrices. It's what the Mirror-apply and Flow-rig bakes ride on,
  and a foundation for future "bake it back out" tools.
- Structured import now covers multi-object static models, so a few kit props (a
  robot arm, a barrel cluster) become pose-able too, at no visual cost.
