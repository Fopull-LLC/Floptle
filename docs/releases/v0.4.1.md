# Floptle v0.4.1 — "Author's Comfort"

A quality-of-life release across the authoring tools: a responsive UI system,
first-class UI shaders with a built-in effect library, drop shadows, native
asset import that works on every OS, and a particle-lifetime option that stops
fighting you. Every item is a generic, public engine feature — the solar demo
is just the first consumer.

(Supersedes the mis-tagged v0.4.0, which shipped without a version bump or notes.)

## UI system — responsive layout & a real canvas scaler (`floptle-ui`)

- **Canvas scaler.** A UI layer now has a `scale_mode` — Match Height (the
  classic default, unchanged), Match Width, Blend W↔H (a slider), Expand (fit /
  letterbox), Shrink (fill / crop), and Constant-pixels — plus a real 2D
  **reference resolution** (width + height) instead of a single height knob. This
  is what keeps a HUD consistent across monitor sizes and aspect ratios.
- **Responsive positioning.** A new **Stretch** placement: a two-point anchor
  (min/max fractions of the parent) with per-side margins. Axes where max > min
  stretch with the parent — e.g. "16 units in from all four edges, at any window
  size"; equal min/max is a point anchor keeping its own size. Quick presets
  (fill all / top / bottom / left / right) in the inspector.
- **Min/max size clamps** on any element so `%`/`fit`/`stretch` elements can't
  collapse on small windows or balloon on ultrawides.

## UI shaders — now first-class, with a built-in effect library

- **The inspector shows it.** Any UI element has an **✨ Effect** section: a
  dropdown of built-in effects, a Custom `.flsl` picker, and **live parameter
  rows** generated from the shader's uniforms (with inline compile errors). No
  more shader-driven elements that look inert in the inspector.
- **Nine built-in effects**, seeded into `shaders/examples/ui/` and each a
  copyable worked example: Outline (pulse), Gradient fill, Gloss sweep, Glow,
  Wobble shimmer, Scanlines (CRT), Holographic, Film grain, and **Frosted glass**.
- **Texture & backdrop sampling.** UI shaders can now read the element's own
  image with `baseTexture(uv)` and — the big one — the **composited scene behind
  the UI** with `backdrop(uv)`, so a panel can frost/refract whatever it's over
  (real backdrop blur, not a fake overlay). Degrades gracefully to a tinted glass
  panel where a capture isn't available.
- **Effects auto-clip to the element's rounded corners** — no more spill past a
  rounded panel's edges.
- **Drop shadow** is built into any shape (color / offset / blur / spread) — a
  soft, feathered shadow behind the panel (the one effect that lives outside the
  element rect, so it's a shape feature rather than a shader).

## Assets — native import that works everywhere

- **⤓ Import…** in the Assets header and **⤓ Import here…** in a folder's
  right-click menu open the OS's native file picker (via the XDG desktop portal),
  so importing works on **Wayland** — where the windowing layer delivers no
  drag-and-drop — as well as X11, Windows and macOS. Multi-select, folders copy
  recursively, names auto-suffix so nothing is clobbered, models register
  immediately, and it runs off the UI thread so the editor never freezes.
- **Drag-and-drop** from the file manager still works on X11/Windows/macOS, with
  more robust routing (a release always lands in a sensible folder).

## Particles — lifetime scale mode (`floptle-vfx`)

- Editing an effect's **lifetime** now offers two behaviors via a per-effect
  option: **Absolute** (Unity-style — automation graphs keep their real-time
  positions; the default, unchanged) or **Scale with lifetime** (Roblox-style —
  the whole timeline stretches/squashes so the shape is preserved over the new
  length). Per-particle life curves stay normalized either way.

## Solar demo — a digital ship HUD & a cleaner cockpit

The first consumer of the new UI shaders. The flight interface went from flat
text scattered on screen to a laid-out cockpit:

- **Framed instrument panels.** The flight readout now sits in a digital HUD
  panel (dark glass, hairline border, drop shadow, a subtle CRT scanline) that
  auto-sizes to its content and appears/vanishes with the readout.
- **Staging moved out of the way.** The stage list was floating in the middle of
  the right edge — it's now a matching panel in the top-right corner. Layout
  reads like a real cockpit: telemetry top-left, staging top-right, navball +
  speed/altitude tapes bottom-center, SAS holds on the left.
- **No more broken glyphs.** Every status icon that the HUD font couldn't render
  (the ▢ boxes) is now an avionics-style flag — `[SOLAR]`, `[LINK]`, `[HEAT]`,
  `[CHUTES OPEN]`, `[DAMAGE]`, `[TANK EMPTY]`, `[ESCAPING]`, `[WARP 4×]`,
  `GEAR DOWN/UP`, `>> BURN NOW`, `[OK]`.
- **Interactive menus & buttons.** Menu / builder / facility panels gained drop
  shadows, frosted-glass and gloss/outline shader accents, and buttons now give
  hover/press feedback.

## Animator (recap of the pre-release polish)

- Dopesheet: box + shift multi-select, move/stretch/copy/cut/paste/duplicate/
  delete, undo/redo, key-all-bones / key-all-tracks, insert-key, track-click to
  select, prev/next-key navigation, and a shortcuts cheat-sheet.
- Bone gizmos in the Scene view with live drag preview; skinned-mesh selection
  outline offset fixed; rig-only glTF characters now expose their bones.
